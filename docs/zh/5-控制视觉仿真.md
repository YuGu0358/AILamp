# 5. 控制、视觉与仿真

仿真：

```bash
ailamp sim-demo
ailamp sim-viewer --render outputs/ailamp.png
```

AILamp 保持原 LeLamp 的仿真流程：

```text
仿真软件：MuJoCo
主场景：simulation/ailamp_scene.xml
原始参考场景：simulation/scene.xml
机器人 MJCF：simulation/robot.xml
参考 URDF：simulation/robot.urdf
网格资产：simulation/assets/*.stl
```

`simulation/ailamp_scene.xml` 包含 LeLamp 机器人、虚拟人物目标、仿真摄像头，以及 AILamp 新硬件适配件的简化可视几何。

场景中的适配件可视对象：

```text
ailamp_jetson_tray_visual -> Jetson Nano 底座托盘
ailamp_electronics_deck_visual -> Waveshare ST3215 驱动板 + Pico WH 侧板
ailamp_camera_mount_visual -> Arducam UB0234 灯头摄像头支架
ailamp_neomatrix_visual -> 扩散片后方的 NeoMatrix 支架
ailamp_respeaker_visual -> ReSpeaker XVF3800 外置支架
```

这些几何体只用于显示安装位置，不改变 LeLamp 舵机运动链或 actuator 映射。

真实硬件：

```bash
ailamp camera-test
ailamp audio-test
ailamp led-test
ailamp vision-demo
ailamp vision-loop --frames 30
ailamp vision-loop --with-outputs
ailamp agent-tools-test --event person_close --apply
ailamp agent-tools-test --event posture_studying --apply
ailamp agent-tools-test --event person_right --offset 0.6 --request "看着我并跟随我" --apply
ailamp agent
```

事件映射：

```text
no_person -> scanning -> 深蓝
person_left -> headshake -> 蓝色
person_center -> nod -> 暖色
person_right -> scanning -> 蓝色
person_close -> shy -> 粉红
person_far -> curious -> 绿色
person_left_seat -> idle -> 深蓝
gesture_left -> headshake -> 蓝色
gesture_right -> scanning -> 蓝色
gesture_up -> curious -> 冷白蓝
gesture_down -> idle -> 冷白蓝
posture_studying -> idle -> 专注暖白
looking_at_lamp -> nod -> 暖色
expression_smile -> happy_wiggle -> 暖色
expression_tired -> idle -> 专注暖白
expression_neutral -> idle -> 冷白蓝
```

`vision-demo` 只抓取一帧，打印识别事件、动作和灯光颜色。

`vision-loop` 是连续运行的真实闭环。

Orin 配置：

```text
Arducam UB0234 -> YOLO nano + YOLO pose -> VisionEvent -> DecisionService -> ST3215 + Pico LED
```

Jetson Nano API-hybrid 配置：

```text
Arducam UB0234 -> 低频 OpenAI vision API -> VisionEvent -> DecisionService -> ST3215 + Pico LED
```

默认只打印结果并写入 `outputs/vision_state.json`。在 Jetson 上确认 `led-test` 和 `motor-test` 通过后，再使用 `--with-outputs` 驱动真实舵机和 LED。

```bash
ailamp --config config/hardware.jetson-nano.toml vision-loop --with-outputs
```

## AI 决策层

AILamp 在视觉/语音输入和硬件输出之间加入了本地决策层：

```text
VisionEvent + 可选语音请求 -> DecisionService -> 动作 recording 或关节增量 + LED
```

连续跟随决策：

```text
person_left / gesture_left -> base_yaw 负向增量，慢慢向左
person_right / gesture_right -> base_yaw 正向增量，慢慢向右
person_close -> wrist_pitch 负向增量，灯头轻微后仰
person_far -> wrist_pitch 正向增量，灯头前倾
gesture_up -> wrist_pitch 正向增量
gesture_down -> wrist_pitch 负向增量
```

语音意图可以覆盖默认视觉反应：

```text
"专注" / "学习" / "focus" / "study" -> idle + 专注暖白灯
"点头" / "nod" -> nod
"跟随" / "看着我" / "follow" / "track" -> 连续跟随
"休息" / "待机" / "idle" / "rest" -> idle
```

关节增量命令在发送给 ST3215 层前会经过安全限幅。

当前手势与姿态支持使用启发式规则：

- 手向左/右/上/下会触发对应方向的灯位调整反应
- 低头学习会开启专注暖白灯光
- 抬头看台灯会触发点头回应
- 人靠近镜头优先触发 `shy`
- 人离开座位会切换到 `idle`

如果希望 AI 工具使用实时摄像头状态，需要同时运行 `vision-loop` 和 `agent`。LiveKit/OpenAI agent 读取同一个 `outputs/vision_state.json`，工具能力包括：

- 描述可用工具
- 根据视觉和语音意图做决策
- 读取当前视觉状态
- 给出当前视觉状态对应的动作和灯光
- 将当前视觉行为应用到真实台灯
- 列出可用动作 recording
- 播放指定动作 recording
- 设置自定义 LED 颜色

`agent-tools-test` 会调用同一套 AI 可用工具，不需要 LiveKit 或真实硬件，默认使用 dry-run 输出：

```bash
ailamp agent-tools-test --event person_close --apply --recording nod --color 1 2 3
ailamp agent-tools-test --event posture_studying --apply
ailamp agent-tools-test --event person_right --offset 0.6 --request "看着我并跟随我" --apply
```
