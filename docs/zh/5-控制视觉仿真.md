# 5. 控制、视觉与仿真

仿真：

```bash
ailamp sim-demo
ailamp sim-viewer --render outputs/ailamp.png
```

AILamp 保持原 LeLamp 的仿真流程：

```text
仿真软件：MuJoCo
主场景：simulation/ailamp_scene.xml
原始参考场景：simulation/scene.xml
机器人 MJCF：simulation/robot.xml
参考 URDF：simulation/robot.urdf
网格资产：simulation/assets/*.stl
```

真实硬件：

```bash
ailamp camera-test
ailamp audio-test
ailamp led-test
ailamp vision-demo
ailamp vision-loop --frames 30
ailamp vision-loop --with-outputs
ailamp agent
```

事件映射：

```text
no_person -> scanning -> 深蓝
person_left -> headshake -> 蓝色
person_center -> nod -> 暖色
person_right -> scanning -> 蓝色
person_close -> shy -> 粉红
person_far -> curious -> 绿色
```

`vision-demo` 只抓取一帧，打印识别事件、动作和灯光颜色。

`vision-loop` 是连续运行的真实闭环：

```text
Arducam UB0234 -> YOLO nano -> VisionEvent -> BehaviorService -> ST3215 + Pico LED
```

默认只打印结果并写入 `outputs/vision_state.json`。在 Jetson 上确认 `led-test` 和 `motor-test` 通过后，再使用 `--with-outputs` 驱动真实舵机和 LED。

如果希望 AI 工具使用实时摄像头状态，需要同时运行 `vision-loop` 和 `agent`。LiveKit/OpenAI agent 读取同一个 `outputs/vision_state.json`，工具能力包括：

- 读取当前视觉状态
- 给出当前视觉状态对应的动作和灯光
- 将当前视觉行为应用到真实台灯
- 播放指定动作 recording
- 设置自定义 LED 颜色
