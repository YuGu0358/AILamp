# 3. 装配

AILamp v1 保留 LeLamp 原始打印件。第一版不要切割 `LampHead.3mf`，也不要切割、钻孔或永久修改原始底座、灯臂、灯头和扩散片。

## 机械适配（v6 架构 — 与原 LeLamp 底座拓扑一致）

AILamp v6 底座完全按原 LeLamp `LampBase.3mf` + `LampBase - Cover.3mf` 的接口拓扑做，只是把外形扩大到能容纳 Jetson Nano。整体逻辑：

- **`AILamp_LampBase_Electronics_Shell.3mf` 是底面开口、顶面封闭的电子舱**。顶面有两个矩形开口：靠后的 24.8 × 45.3 mm 是 base_yaw 电机本体过孔（XY 位置 0, −25.7）、靠前的 24.7 × 10 mm 是日链走线槽（XY 位置 0, +4.9）。壳体内部从顶面挂下来一个 30.8 × 51.3 mm 的电机舱，把 STS3215 本体卡在里面，舵盘从顶面过孔伸出来朝上。
- **`AILamp_LampBase_Electronics_Cover.3mf` 是底板**。188 × 228 × 6 mm，4 mm 唇 + 2 mm 凸缘，从下往上扣进壳体的开口底，4 个角的 M2.5 螺丝把它锁到壳体内部的 4 根全高螺柱上。
- 原 LeLamp **`LampBase.3mf` 和 `LampBase - Cover.3mf` 不打印**，只留作参考。AILamp 用上面两件替代。

装配顺序：

1. 完成 LeLamp 原始灯臂（Base-Elbow / Elbow-Wrist / Pitch ×2）、腕部、灯头和扩散片装配，不改尺寸。
2. 把 **STS3215 (ID = 1, base_yaw)** 从下方推入 `AILamp_LampBase_Electronics_Shell` 顶面内部的 30.8 × 51.3 mm 电机舱，舵盘端朝上从顶面 24.8 × 45.3 mm 矩形孔伸出来。**先不要给 base_yaw 装舵盘** —— 灯臂 Base-Elbow 内部的舵盘锁位是 4 × M2 在 Φ14 mm 螺孔分度圆上（厂家工程图 #HornScrewDistDia = 14 mm），把舵盘提前装上反而会挡住灯臂下罩。
3. 把灯臂 **Base-Elbow** 从上方往下罩到电机本体上，让灯臂底部的开口电机笼正好包住伸出来的 STS3215 本体。这时舵盘还没装，灯臂可以自由摆放。
4. 在灯臂内部把舵盘对准电机轴推到位，用 4 × M2 螺丝从灯臂内部把舵盘锁到灯臂上 —— 这是 LeLamp 原版的步骤，AILamp 不重新设计这个接口。
5. 把日链电源/信号线从灯臂内部下来，穿过壳体顶面 24.7 × 10 mm 走线槽进入电子舱。
6. 把 **Jetson Nano** 放到内部前部 84 × 64 mm 螺柱阵列（板中心位 0, +50）；**Waveshare ST3215 驱动板**放到内部更前的 58 × 23 mm 螺柱阵列（板中心位 −42, +110）。两者都从顶面挂下来 8 mm / 7 mm 的螺柱顶端固定 —— 不在底部、是因为 v6 壳体底部本来就是开的，所有内部螺柱都是全高柱（z = 0 → 39 mm）双端支撑。
7. 把 **Pico WH** 放进侧板 deck 或用扎带固定，保持 USB Type-A 朝向后壁的开窗。
8. 走线（USB、电源、舵机总线、LED）确认松紧、不顶死板边后，再上 **`AILamp_LampBase_Electronics_Cover.3mf`** 从下方扣进壳体的开口底。
9. 用 4 × M2.5 自攻螺丝从底板下方穿过盖板 6 mm 厚度，锁进壳体内部 4 个角的 (±82, −88) 和 (±82, +118) mm 全高螺柱。
10. 可选适配件：`AILamp_Cable_Clip_6mm.3mf` 用于 USB/信号线、`AILamp_Cable_Clip_10mm.3mf` 用于电源/舵机线；`AILamp_Jetson_Nano_Base_Tray.3mf` 和 `AILamp_Electronics_Side_Deck.3mf` 是台架调试用，可以把硬件放在台灯外面接线先跑通再装回。

尺寸规则：v6 不靠卡扣压板子，全部用螺丝。盖板和壳体的配合是 0.5 mm 唇间隙；电机舱内壁和 STS3215 本体是 0.05 mm 每侧 slack。试打 0.1 mm 层高、20% infill 没问题。

不要把未修改的 LeLamp 原版 `LampBase.3mf` 当作 Jetson Nano 的封闭电子仓 —— 原版底座是给 Raspberry Pi 设计的，给 Jetson Nano 完全装不下。AILamp shell 把外形扩大到 190 × 230 × 42 mm，但电机舱位置和走线槽位置都对齐到原版 LampArm Base-Elbow 期望的 (0, −25.7) / (0, +4.9)，所以原版灯臂可以照搬不改。

## 接线

```text
Jetson USB -> Waveshare Servo Driver with ESP32 -> 5x Waveshare ST3215
Jetson USB -> Raspberry Pi Pico WH -> TXS0108E -> NeoMatrix DIN
Jetson USB -> Arducam UB0234
Jetson USB -> ReSpeaker XVF3800
GST120A12-P1J -> 急停开关 -> DC barrel terminal -> WAGO 221-413 -> 舵机驱动板供电
GST60A05-P1J -> DC barrel terminal -> WAGO 221-413 -> NeoMatrix 5V/GND
需要信号连接的设备按接口要求共地
```

12V 舵机供电域和 5V LED 供电域要物理分开。摄像头和音频 USB 线尽量远离舵机电源线。

## LED 保护

```text
Pico GP0 -> TXS0108E A 侧输入
Pico 3V3 -> TXS0108E VCCA
NeoMatrix 5V -> TXS0108E VCCB
公共 GND -> TXS0108E GND
TXS0108E B 侧输出 -> 330 ohm resistor -> NeoMatrix DIN
TXS0108E OE -> VCCA
GST60A05-P1J 5V/GND -> 1000 uF capacitor -> NeoMatrix 5V/GND
```

## 上电前检查

软件驱动真实硬件前先确认：

1. 急停开关能断开 12V 舵机电源。
2. `GST120A12-P1J` 只给舵机驱动板和 ST3215 链路供电。
3. `GST60A05-P1J` 只给 NeoMatrix LED 系统供电。
4. Pico、TXS0108E 和 NeoMatrix 已共地。
5. 每个 ST3215 都已经设置为预期 ID，再把台灯放入空间受限的姿态。

不要在 `led-test` 和 `motor-test` 通过前运行 `--with-outputs`。

## 首次启动

Jetson Nano API-hybrid 主流程：

```bash
ailamp --config config/hardware.jetson-nano.toml runtime-check
ailamp --config config/hardware.jetson-nano.toml hardware-check --include-devices
ailamp --config config/hardware.jetson-nano.toml camera-test
ailamp --config config/hardware.jetson-nano.toml audio-test
ailamp --config config/hardware.jetson-nano.toml led-test
ailamp --config config/hardware.jetson-nano.toml motor-test
```

默认 Orin profile 或快速 smoke check 可使用旧命令 `ailamp hardware-check`、`ailamp led-test`、`ailamp vision-demo`、`ailamp agent`。在 Jetson Nano 上，以带 `--config config/hardware.jetson-nano.toml` 的流程为准。

## 软件验收

真实运动前先运行 dry-run 软件检查：

```bash
ailamp --config config/hardware.jetson-nano.toml agent-tools-test --event person_right --offset 0.6 --request "看着我并跟随我"
ailamp --config config/hardware.jetson-nano.toml vision-loop --frames 30
```

确认台灯有足够物理活动空间后，再运行真实输出：

```bash
ailamp --config config/hardware.jetson-nano.toml vision-loop --with-outputs
ailamp --config config/hardware.jetson-nano.toml agent --with-outputs
```
