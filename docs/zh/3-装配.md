# 3. 装配

AILamp v1 保留 LeLamp 原始打印件。第一版不要切割 `LampHead.3mf`，也不要切割、钻孔或永久修改原始底座、灯臂、灯头和扩散片。

## 机械适配

1. 先完成 LeLamp 原始灯臂、腕部、灯头和扩散片装配。`LampBase.3mf` 和 `LampBase - Cover.3mf` 只作为参考，不作为 AILamp 最终底座打印件。
2. 将 `AILamp_LampBase_Electronics_Shell.3mf` 作为替换底座壳体。确认它给 Jetson Nano 4GB 高度、USB 插头、DC 电源插头、摄像头/音频线和线缆弯折半径都留出空间。
3. 将 `AILamp_LampBase_Electronics_Cover.3mf` 作为替换底座上盖。确认凸起灯臂固定座能包围底座舵机/灯臂根部且不摩擦，并在上盖和灯臂之间形成可见的机械过渡。
4. 将 `AILamp_Base_Arm_Link_Boot.3mf` 安装在固定上盖凸台和运动灯臂根部之间。确认它在仿真中跟随灯臂根部 transform，不会和固定上盖凸台卡住。
5. 用 x 方向 +/-82 mm、y 方向 -88 mm 和 +118 mm 的四角螺丝孔位连接上盖和壳体。先不要完全拧紧，等线缆走线确认后再锁紧。
6. 将 Jetson Nano 固定到内部 84 x 64 mm 定位柱孔位，再把 Pico WH 和 Waveshare ST3215 驱动板放进控制板区域。Waveshare 驱动板定位柱使用 58 x 23 mm 固定孔距。
7. 确认板子可以取出，不会压弯排针，也不会挡住 USB-C、Micro-USB、舵机总线、12V 输入或 5V LED 线。
8. `AILamp_Cable_Clip_6mm.3mf` 用于底座附近的 USB/信号线，`AILamp_Cable_Clip_10mm.3mf` 用于电源线或舵机线束。
9. `AILamp_Jetson_Nano_Base_Tray.3mf` 和 `AILamp_Electronics_Side_Deck.3mf` 是可选台架调试件，用于把硬件放在台灯外面调试。

尺寸规则：第一版适配件宁可稍松，不要卡死。确认板子能装入后，再用螺丝、扎带或可拆绑带固定。

不要把未修改的 LeLamp 原始底座当作 Jetson Nano 的封闭内部电子仓。AILamp 壳体和上盖会替代原底座；建议先低填充试打并检查插头空间，再最终打印。

## 接线

```text
Jetson USB -> Waveshare Servo Driver with ESP32 -> 5x Waveshare ST3215
Jetson USB -> Raspberry Pi Pico WH -> TXS0108E -> NeoMatrix DIN
Jetson USB -> Arducam UB0234
Jetson USB -> ReSpeaker XVF3800
GST120A12-P1J -> 急停开关 -> DC barrel terminal -> WAGO 221-413 -> 舵机驱动板供电
GST60A05-P1J -> DC barrel terminal -> WAGO 221-413 -> NeoMatrix 5V/GND
需要信号连接的设备按接口要求共地
```

12V 舵机供电域和 5V LED 供电域要物理分开。摄像头和音频 USB 线尽量远离舵机电源线。

## LED 保护

```text
Pico GP0 -> TXS0108E A 侧输入
Pico 3V3 -> TXS0108E VCCA
NeoMatrix 5V -> TXS0108E VCCB
公共 GND -> TXS0108E GND
TXS0108E B 侧输出 -> 330 ohm resistor -> NeoMatrix DIN
TXS0108E OE -> VCCA
GST60A05-P1J 5V/GND -> 1000 uF capacitor -> NeoMatrix 5V/GND
```

## 上电前检查

软件驱动真实硬件前先确认：

1. 急停开关能断开 12V 舵机电源。
2. `GST120A12-P1J` 只给舵机驱动板和 ST3215 链路供电。
3. `GST60A05-P1J` 只给 NeoMatrix LED 系统供电。
4. Pico、TXS0108E 和 NeoMatrix 已共地。
5. 每个 ST3215 都已经设置为预期 ID，再把台灯放入空间受限的姿态。

不要在 `led-test` 和 `motor-test` 通过前运行 `--with-outputs`。

## 首次启动

Jetson Nano API-hybrid 主流程：

```bash
ailamp --config config/hardware.jetson-nano.toml runtime-check
ailamp --config config/hardware.jetson-nano.toml hardware-check --include-devices
ailamp --config config/hardware.jetson-nano.toml camera-test
ailamp --config config/hardware.jetson-nano.toml audio-test
ailamp --config config/hardware.jetson-nano.toml led-test
ailamp --config config/hardware.jetson-nano.toml motor-test
```

默认 Orin profile 或快速 smoke check 可使用旧命令 `ailamp hardware-check`、`ailamp led-test`、`ailamp vision-demo`、`ailamp agent`。在 Jetson Nano 上，以带 `--config config/hardware.jetson-nano.toml` 的流程为准。

## 软件验收

真实运动前先运行 dry-run 软件检查：

```bash
ailamp --config config/hardware.jetson-nano.toml agent-tools-test --event person_right --offset 0.6 --request "看着我并跟随我"
ailamp --config config/hardware.jetson-nano.toml vision-loop --frames 30
```

确认台灯有足够物理活动空间后，再运行真实输出：

```bash
ailamp --config config/hardware.jetson-nano.toml vision-loop --with-outputs
ailamp --config config/hardware.jetson-nano.toml agent --with-outputs
```
